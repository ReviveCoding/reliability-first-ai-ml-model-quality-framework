# Security Policy

## Supported version

This repository is a research and portfolio framework. Security fixes are applied to the latest commit on the `main` branch.

| Version | Supported |
|---|---|
| Latest `main` | Yes |
| Older commits or forks | No |

## Reporting a vulnerability

Please use GitHub private vulnerability reporting when it is enabled for this repository.

Do not include exploit details, credentials, personal data, or sensitive datasets in a public issue. If private reporting is unavailable, open a minimal issue requesting a private contact channel without disclosing technical details.

## Scope

Security reports may include:

- exposed credentials or sensitive paths;
- unsafe deserialization or arbitrary-code execution;
- dependency or workflow supply-chain risks;
- data leakage between training, selection, and test stages;
- unintended publication of private or proprietary data.

Model-quality disagreements, expected benchmark limitations, and feature requests should use normal GitHub issues rather than the security channel.
